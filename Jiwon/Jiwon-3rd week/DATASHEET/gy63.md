I2C
-------------
- RESET SEQUENCE
	- Master에서는 시작신호와 디바이스 주소와 쓰기를 한번에 보낸다. 또한 cmd byte에  리셋 명령을 보내는 일을 한다. 멈추라는 신호를 보낸다
	- Slave에서는 Master 보내준 신호,명령을 잘 받았다는 응답을 해준다.

- PROM READ SEQUENCE
	- 첫번째 명령 : 시스템을 PROM 읽기 모드로 설정
		- Master에서는 시작신호와 디바이스 주소와 쓰기를 한번에 보낸다. 또한 cmd byte에  시스템을 PROM 읽기 모드로 설정하는 명령을 보내는 일을 한다. 멈추라는 신호를 보낸다.
		- Slave에서는 Master에서 보낸 명령을 잘 받았다는 응답을 한다.

	- 두번째 명령 : 시스템에서 데이터를 가져온다.
		- Master에서는 시작신호,디바이스 주소,읽기를 한번에 보낸다 또한 Slave에서 보낸 것을 자 받았다고 응답하고 못받았다고 또한 응답한다. 그리고 멈추는 신호를 보낸다.
		- Slave에서는 Master에서 보내준 명령을 잘 받았다고 응답해주며 Memort bit를 가져 온다.

- CONVERSION SEQUENCE
	- I2C Command to initiate a pressure conversion
		- Master에서는 한번에 시작과 디바이스 주소 쓰기를 해준다. 전환 명령을 보낸다.
멈추는 신호 보냄
		- Slave에서는 승인을 해준다.
	
	- I2C ADC read sequence
		-  Master에서는 시작과 디바이스 주소,쓰기를한번에 한다. 읽기 명령 보낸다. 멈춤 신호 보낸다.
		- Slave에서는 승인을 해준다.
	- I2C answer from MS5611-01BA
		- Master에서는 시작과 디바이스 주소 읽기를 한다.  Slave 승인하거나 승인하지 않는다 멈춤 신호 보낸다.
		- Slave에서는 24SCLK 사이클을 보낸다. Master에 승인 신호 보낸다.

- I2C 통신이란?
	- 데이터를 주고 받기 위한 선(SDA) 하나와 송수신 타이밍 동기화를 위한 클럭 선(SCL) 하나로 이루어진다.

- I2C 통신의 특징
	- 2개의 선을 이용하는 통신 방식
	- 하나의 마스터에 여러 개의 슬레이브 기기가 연결 가능
	- 클럭 신호 사용하는 동기식 통신 방식이기 때문에 시간에 자유로움 	

SPI
-------------

- RESET SEQUENCE		
	- 내부 레지스터 안에있는 불러온 prom을 캘리브레이션 하기위해서 전원을 킨 후에 한번 리셋 시퀀스를 보내주어야한다. 알 수 없는 상태에서 디바이스 ROM을 리셋할 때 사용할 수 있다.

- CONVERSION SEQUENCE 
	- 전환 명령은 고정되지 않은 압력(D1) 또는 고정되지 않은 온도(D2) 변환을 준비하는데 사용된다. 칩 선택은 이 시간 동안 다른 디바이스 장치와의 의사소통을 할 수 없도록 할 수 있다.
전환 후 , ADC 읽기 명령을 사용하면 결과가 최상위 비트와 출력된다. 만약 ADC 읽기 명령 전에 전환을 실행하지 않는다거나 ADC 읽기 명령이 반복된다면 그것은 output  result에 0을 주는것이다. 만약 ADC 읽기 명령 전환 동안에 전송하면 결과는 0이 될 것이고, 전환이 멈추지 않고 마지막 결과가 잘못 될 것이다. 이미 시작된 전환 프로세스 중에 전송된 전환 시퀀스 역시 잘못된 결과가 나온다.

- PROM READ SEQUENCE
	- 읽기 명령은 캘리브레이션 PROM 내용을 읽고 캘리브레이션 계수를 계산할 때 PROM을 사용자에 의해 리셋 후 한번 실행한다. 총 8개의 주소가 있고 이로 인해 총 메모리는 128 bit이다.
주소 0은 공장 테어터 그리고 셋업을 포함하며, 주소 1-6 캘리브레이션 계수이고 주소 7은 시리얼 코드 그리고 순환 중복 검사를 포함한다. 명령 시퀀스는 8비트 길이의 최상위 비트가 먼저 출력되는 16비트 결과 입니다.


- SPI 통신이란?
	- 하나의 Master와 하나 또는 다수의 Slave Device간의 통신 방식
	- 외부 주변장치와 Clock을 통하여 동기화 하는 동기식 통신 방식

-SPI 통신 장점
	- 전이중 통신 : 자신이 말하며 자신이 들을 수 있다.
	- 전송되는 비트에 대한 프로토콜 유연성 : 최대 16비트까지 마음대로 길이 조작 가능
	- 전송기가 필요 없다.
	- 단순한 하드웨어 인터페이스 처리 : 단순한 센서나 메모리에서 많이 사용

- SPI 통신 단점
	- 하드웨어 슬레이브 인식 없다.
	- 슬레이브에 의한 하드웨어 흐름제어가 없다.
	-하나의 마스터 장치만 지원
