Union
=============

- 여러 테이블들을 결합시키는 것에 대해서는 	‘조인(Join)’과 비슷한 개념이지만, ‘Union’은 테이블들을 수직으로 결합시킨다. 
- 조인(Join) 이란?
	두 개 이상의 테이블을 서로 묶어서 하나의 결과 집합으로 만들어 내는 것

- 구조체와 Union의 차이점
	*메모리를 활용 방법의 차이
	- 구조체는 멤버 변수 마다 각각의 메모리를 	할당해주지만, Union은 멤버 변수 중 가장 	메모리 할당량이 큰 변수 하나의 공간만 	할당되어 그 메모리 공간을 서로 공유

- Union의 특징
	- Union의 멤버로 선언된 변수들은 메모리 어드레스를 공유하게 된다


- Union 주의 사항
	- 각 SELECT문의 열 수는 같아야 한다.
	- 각 SELECT문의 집계 함수는 같아야 한다.
	- SELECT문의 순서는 결과에 영향을 미치지 않는다.
	- Union의 결과 중복 값은 하나의 행으로 나온다.
 (중복 값이 보고 싶다면 UNION ALL을 사용)
	- ORDER BY는 Union의 맨 끝에 하나만 받을 수 있다.
- Union 사용
	- 시리얼 통신 또는 TCP/IP 통신에 사용
- Union의 사용 이유
	- 특정 공간을 효율적으로 관리 하기 위하여 사용한다.
